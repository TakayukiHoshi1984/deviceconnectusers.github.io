WebSocketについて

ここでは、GotAPI 1.1に対応したイベント通知系のAPIを確認する方法を説明します。

DeviceConnectのWeb APIを使う際に、幾つかイベント通知系のAPIがあります。これらを利用する際にはWebSocketを使って接続しておく必要があります。

エンドポイントについて

  • パス
    /gotapi/websocket
  • プロトコル
    WebSocket

便利なツール

Node.jsではコンソールからWebSocketサーバに接続できるwscatコマンドが利用できます。

npm install -g wscat

でインストールできます。

接続してみる

GotAPI 1.1では、WebSocket経由でもアクセストークンの送信が必要となりました。事前にアクセストークンを取得してください。ただし、Local OAuthをOFFにしている場合は、任意の文字列を指定可能です。

例えばDeviceConnect Managerが 192.168.0.20 で動作している場合、

$ wscat -c ws://192.168.0.20:4035/gotapi/websocket -o "Null"

で接続できます。接続したらアクセストークンを送信します。ただし、Local OAuthをOFFにしている場合は、任意の文字列で構いません。

> {"accessToken": "16f...5af"}
>

正常に受理された場合は、下記のように成功応答が返ってきます。

< {"result": 0}

これでWebSocket側は準備完了です。

Web APIをコールする

次にWeb API側を準備します。例えば[バッテリーステータス変更API]をコールします。

$ curl -H "Origin: Null" -X PUT "http://192.168.0.20:4035/gotapi/battery/onchargingchange?serviceId=Host...org&accessToken=16f...5af"
{"result":0,"product":"Device Connect Manager","version":"2.0.0"}

注意: GotAPI 1.1では、セッションキー（sessionKey）は不要となります。

イベントを受け取る

後はバッテリーを抜き差ししてイベントがWebSocket側に通知されればOKです。

< {"battery":{"charging":false},"profile":"battery","serviceId":"Host.e87...org","attribute":"onchargingchange"}
< {"battery":{"charging":true},"profile":"battery","serviceId":"Host.e87...org","attribute":"onchargingchange"}
